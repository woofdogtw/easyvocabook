# Rust Desktop App 探索

## 技術棧決策

- **UI framework**：iced 0.14.0（2024-12 發布）
- **SQLite**：rusqlite 0.40.1（不用 sqlx，理由見下）
- **平台路徑**：dirs 6.0.0（`data_local_dir()`）
- **測試用臨時 DB**：tempfile 3.27.0
- **Async runtime**：啟用 iced 的 `tokio` feature，DB call 用 `tokio::task::spawn_blocking` 包 rusqlite，網路 sync 用 async crate

### 為何用 rusqlite 不用 sqlx
- sqlx compile-time query 檢查需要 build 時有 DB 或 `.sqlx` 快取，CI 複雜
- sqlx 綁 tokio，但我們已透過 iced `tokio` feature 引入 tokio，無需重複
- 單一本機 SQLite 檔，無需 connection pool；rusqlite 更簡單、binary 更小
- 長時間操作（sync）用 `spawn_blocking` 包 rusqlite，不卡 UI，效果等同 async

### Crate 結構
- 單一 crate（不用 workspace），db/ui/network 以模組拆分

## DB Schema

### 設計原則
- 單一 DB 檔案，固定名稱 `easyvocabook.db`
  - Windows：`%LOCALAPPDATA%\easyvocabook\`
  - Linux：`~/.local/share/easyvocabook/`
  - Android：`filesDir`
- 同一本書可混放多語言單字，`language` 欄位在 word 層級（ISO 639-1）
- 不設手動熟悉度（星星），改用 `practice_count` / `correct_count` 自動計算
- 設定頁提供「清除測驗計數」功能

### Schema

```sql
-- 書籍元資料
CREATE TABLE db_info (
    name             TEXT    NOT NULL,
    description      TEXT,
    default_language TEXT    NOT NULL DEFAULT 'en',
    version          INTEGER NOT NULL,
    last_modified    INTEGER NOT NULL   -- Unix epoch (i64, seconds)
);

-- 核心單字表
CREATE TABLE words (
    id             INTEGER PRIMARY KEY AUTOINCREMENT,
    word           TEXT    NOT NULL,
    reading        TEXT,               -- 讀音/假名，可空
    meaning        TEXT    NOT NULL,   -- 主要中文翻譯（列表顯示用，必填）
    part_of_speech TEXT,               -- 詞性（下拉選單 key，如 noun/verb/i-adj…）
    note           TEXT,               -- 補充說明（自由文字，可空）
    language       TEXT    NOT NULL,   -- ISO 639-1
    practice_count INTEGER NOT NULL DEFAULT 0,
    correct_count  INTEGER NOT NULL DEFAULT 0,
    created_at     INTEGER NOT NULL,   -- Unix epoch (i64, seconds)
    practiced_at   INTEGER            -- Unix epoch (i64, seconds)，可空
);

-- 次要意思（0..N，與 words.meaning 共同構成一個字的全部意思）
CREATE TABLE word_meanings (
    id      INTEGER PRIMARY KEY AUTOINCREMENT,
    word_id INTEGER NOT NULL REFERENCES words(id) ON DELETE CASCADE,
    meaning TEXT    NOT NULL,
    UNIQUE(word_id, meaning)   -- 防止同一字重複輸入相同意思
);

-- 語言特定的附加形態（活用形、格變化、聲調等）
CREATE TABLE word_forms (
    id      INTEGER PRIMARY KEY AUTOINCREMENT,
    word_id INTEGER NOT NULL REFERENCES words(id) ON DELETE CASCADE,
    label   TEXT    NOT NULL,
    value   TEXT    NOT NULL
);

-- 例句
CREATE TABLE sentences (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    word_id     INTEGER NOT NULL REFERENCES words(id) ON DELETE CASCADE,
    sentence    TEXT    NOT NULL,
    translation TEXT                -- 中譯，可空
);
```

### 隨機測驗加權演算法

```
if practice_count == 0:
    weight = NEW_WEIGHT          ← 新字固定高權重（如 3.0）

else:
    incorrect_rate = (practice_count - correct_count) / practice_count
    weight = base + incorrect_rate × multiplier
             ← base=1.0, multiplier=3.0
```

所有詞條共用同一個加權隨機池（不分新/舊桶），權重範例：
- 新字：3.0
- 100% 錯：4.0（略高於新字，出錯嚴重者最優先）
- 50% 錯：2.5
- 全對：1.0（仍在池中，偶爾出現，不會完全消失）

設計意圖：大量新字出現時，出錯率高的舊字仍能被抽到；
具體常數（NEW_WEIGHT / base / multiplier）於 propose 時定案。

小樣本行為：練 1 次就答錯（rate=1.0）與練 50 次全錯的權重相同——這是**刻意設計**，
符合「剛答錯立刻再考」的複習邏輯，不視為 bug。
若未來要讓權重隨樣本累積才穩定，可考慮 Laplace 平滑（α=1），留待 SRS 升級時評估。

## 語言別建議 word_forms 標籤

DB schema 保持簡單通用。App 層面根據語言 + 詞性提供建議 label 清單，使用者可選填或自訂。

### 英文（`language = "en"`）

**通用（所有詞性）**
| label | 說明 | 範例 |
|-------|------|------|
| phonetic | IPA 發音（同音字搜尋用，選填） | fɪʃɪŋ（fishing/phishing）|

**動詞（v.）**
| label | 說明 | 範例 |
|-------|------|------|
| past | 過去式 | went, ran |
| past_participle | 過去分詞 | gone, run |
| gerund_double | 現在分詞重複字母 | true（run→running）|
| transitive | 及物/不及物 | transitive / intransitive / both |

**名詞（n.）**
| label | 說明 | 範例 |
|-------|------|------|
| plural | 不規則複數 | children, feet |
| countable | 可數性 | countable / uncountable |

**形容詞（adj.）**
| label | 說明 | 範例 |
|-------|------|------|
| comparative | 比較級 | better, more beautiful |
| superlative | 最高級 | best, most beautiful |

**副詞（adv.）**
| label | 說明 | 範例 |
|-------|------|------|
| base_adj | 對應形容詞（規則 -ly 衍生） | quick（for quickly）|
| irregular | 不規則副詞 | true（well, fast, hard）|

**介係詞（prep.）**
| label | 說明 | 範例 |
|-------|------|------|
| collocation | 常見搭配動詞/形容詞 | interested IN, responsible FOR |

**片語動詞（phrasal v.）**
介係詞或副詞小品詞構成的動詞片語，如 look up、give in。
| label | 說明 | 範例 |
|-------|------|------|
| past | 過去式 | gave in |
| past_participle | 過去分詞 | given in |
| separable | 可分離 | true（look the word up）/ false（come across）|
| transitive | 及物/不及物 | transitive / intransitive |

**連接詞（conj.）**
通常無特殊活用，word_forms 可留空或記錄用法分類。

**感嘆詞（interj.）**
無特殊活用。

### 日文（`language = "ja"`）

**動詞（v.）**
| label | 說明 | 範例 |
|-------|------|------|
| group | 動詞分類 | 1（五段）/ 2（一段）/ 3（変格）|
| transitive | 自他 | 自動詞 / 他動詞 |
| te_form | て形 | 食べて、書いて |
| ta_form | た形（過去） | 食べた、書いた |
| nai_form | ない形（否定） | 食べない、書かない |
| masu_form | ます形（敬體） | 食べます、書きます |
| particle | 主要助詞 | を / に |
| pitch_accent | 聲調（NHK式數字） | 0、1、2 |

**い形容詞（adj-i）**
| label | 說明 | 範例 |
|-------|------|------|
| te_form | て形 | 大きくて |
| nai_form | ない形 | 大きくない |
| ta_form | 過去形 | 大きかった |
| pitch_accent | 聲調 | 3 |

**な形容詞（adj-na）**
| label | 說明 | 範例 |
|-------|------|------|
| te_form | て形 | きれいで |
| nai_form | ない形 | きれいじゃない |
| pitch_accent | 聲調 | 0 |

**名詞（n.）**
| label | 說明 | 範例 |
|-------|------|------|
| pitch_accent | 聲調（NHK式：0=平板、1=頭高…） | 1（あめ→雨）|
| counter | 助數詞 | 枚、本、匹 |

**片假名外來語（katakana loanwords）**
| label | 說明 | 範例 |
|-------|------|------|
| origin | 原始外來語 | software（for ソフト）、television（for テレビ）|

**漢字讀音顯示**
- `reading` 欄位存平假名讀音（e.g., 食べる → たべる）
- UI 負責渲染漢字上方的振り仮名（furigana）
- 純平假名或片假名單字：`reading` 可空或同 `word`

**自他動詞對應**
日文自他動詞常成對出現（開ける↔開く、出す↔出る），用 `word_forms` TEXT 記錄。
不用 FK（避免刪除詞條時的連鎖問題，且對應字不一定在 DB 裡）。

| label | 說明 | 範例 |
|-------|------|------|
| transitive_pair | 對應他動詞 | 開ける（for 開く）|
| intransitive_pair | 對應自動詞 | 開く（for 開ける）|

- **語言**：UI 顯示語言（繁中 / 簡中 / English），**即時切換**（app state 存 current language，`view()` 每次重繪時套用，無需重啟）

- **同步方式**：radio 選擇，僅展開選中方式的設定欄位（其他收折）
- **FTP/FTPS/SFTP**：host/port/username/directory 存 local config；password 存 OS keychain（`keyring` crate）
- **Google Drive / OneDrive**：OAuth token 存 OS keychain；「登入」按鈕觸發系統瀏覽器 PKCE 流程；已登入則顯示帳號 email + [登出]，不需要「記住登入資訊」checkbox；資料夾以**名稱**輸入（找不到自動建立），內部用 folder ID 操作

- **關於**：固定 footer（不捲動），顯示版本號 + MIT copyright
- **新增單字的預設語言**：不在設定頁，記住上次選的語言（local config，預設 `en`）
- `db_info.default_language`：保留 schema 欄位但不做成設定項，作為初始 fallback
- iced 實作：`Column` 內 `Scrollable`（App/Sync/Practice sections）+ 固定 footer Row

### 搜尋與篩選架構

採**全記憶體快取**方案（不用 debounce + SQL）：

- App 啟動時將**完整聚合**載入 `DbTableMemory`：每個 word 條目包含 words + 所有 word_meanings + 所有 word_forms + 所有 sentences（子表一次載入，不另打 SQL）
- 文字搜尋涵蓋：`word`、`reading`、`meaning`（主要）、所有 `word_meanings.meaning`（次要）
- UI 的篩選（語言下拉 + 文字搜尋）直接在記憶體操作，即時無延遲
- 寫入流程：SQLite 確認寫入 → 更新 DbTableMemory → UI 刷新
- UI 永遠從 DbTableMemory 讀取，SQLite 是 source of truth
- 編輯 dialog 載入時直接從 DbTableMemory 取完整聚合，無需額外 SQL

```
UI ──filter──▶ DbTableMemory ◀──load/update── DbTableSQLite
                  (cache)                      (source of truth)
```

5000 個單字 × ~500 bytes ≈ 2.5MB，記憶體完全不是問題。
`DbTableMemory` 此處就是它的核心存在意義（非只是測試用）。

## 測驗模式設計

### 共用規則
- 同義詞定義：兩個詞條的意思集合（`words.meaning` + `word_meanings`）有任一交集者互為同義詞
- 選擇題干擾項：從其他詞條隨機抽，最多 4 個（含正解），不足時有幾個出幾個（不補滿）；**必須排除「任一意思與正解意思集合有交集」的條目**，避免同義詞混入干擾
- 日文批改：漢字或假名任一邊正確即算對；公布答案時兩者都顯示
- **practice_count / correct_count 更新時機**（統一）：
  - 翻面卡：按「答對」→ practice_count++, correct_count++；按「答錯」→ practice_count++

  - 選擇題：送出後批改，選中集合完全符合正解集合 → practice_count++, correct_count++；否則 → practice_count++

### 出題模式

**中翻英**
- 提示：從該字所有意思（主要 + 次要）**隨機取一個**顯示
- 比對：接受 DB 中任何與該提示意思有交集的詞條之原文（同義詞皆可）
- 公布答案：顯示使用者作答 + 列出所有同義詞
- 動詞同時考 4 格：現在式、過去式、過去分詞、進行式（嚴格 exact match）
- 有 collocation 資料者（時間名詞介係詞等）隨機考

**英翻中**
- 顯示英文 word，複選題：正解集合 = 該詞條的 `words.meaning` + 所有 `word_meanings.meaning`（可能 1..N 個）
- 選項最多 4 個（含所有正解 + 隨機干擾項），不足時有幾個出幾個
- 干擾項抽取時排除「任一意思與正解集合有交集」的其他詞條

**中翻日**
- 提示：從該字所有意思**隨機取一個**顯示；比對與公布規則同中翻英
- 動詞同時考 3 格：辞書形（る）、ます形、た形
- 形容詞：填入原形 + radio 選 い/な 類型，再填活用形
  - 選 い → 考 て形（大きくて）、否定（大きくない）、過去（大きかった）
  - 選 な → 考 て形（きれいで）、否定（きれいじゃない）
  - 類型選擇動態顯示對應欄位（UI state driven by radio selection）
- 有助詞資料（particle）者需同時填助詞
- 接受漢字或假名，公布答案時兩者都顯示

**日翻中**
- 顯示日文 word，複選題，規則同英翻中

### 降頻出題模式（較少出現）
- **例句填空（Cloze）**：需有 `sentences.translation`；用句子遮蔽單字讓使用者填入
- **假名→漢字**：電腦 IME 易作答，降低頻率
- **活用 drill**：隨機指定一個活用形考（e.g., 「食べる」的ます形是？）
- **介係詞/搭配詞 drill**：需有 `word_forms.collocation` 或 `phonetic` 資料

### 待確認
- 出題方向：App 隨機決定，愈隨機愈好（腦力激盪）
  - 依詞條 language 決定可用的方向（英文詞 → 中翻英 或 英翻中；日文詞 → 中翻日 或 日翻中）
  - 降頻模式（填空、活用 drill 等）在有資料時隨機混入，但出現頻率較低
  - 語言篩選下拉（全部 / 英文 / 日文）仍保留，讓使用者選擇今天只練哪個語言

## Network / Sync

### 同步策略
- **整檔同步**（whole-file replace），沿用 Easy 系列做法
- 衝突判斷：比較 `db_info.last_modified`（Unix epoch），讓使用者選擇保留本地或遠端
- AUTOINCREMENT id 在整檔同步模型下安全（不做 record-level merge）

### 支援的 Sync 方式
- FTP / FTPS / SFTP：帳密輸入，無 OAuth，最簡單
- Google Drive：OAuth2 PKCE flow（見下）
- OneDrive：OAuth2 PKCE flow（見下）

### PC (Rust) OAuth 流程
```
1. tokio 起臨時 HTTP server 監聽 http://127.0.0.1:<random_port>/callback
2. webbrowser crate 開系統瀏覽器至授權頁（含 redirect_uri）
3. 使用者在瀏覽器登入授權
4. OAuth provider redirect 回 http://127.0.0.1:<port>/callback?code=XXXXX
5. App 接住 code → 換取 access token + refresh token
6. 瀏覽器顯示「登入成功，可關閉此頁」
```
- Rust crates：`webbrowser`、`oauth2`、tokio HTTP（已引入）
- 不需要 iced WebView

### Android (Kotlin) OAuth 流程
```
1. AndroidManifest.xml 註冊 custom URL scheme：
   tw.idv.woofdog.easyvocabook://oauth2callback
2. Chrome Custom Tabs 開啟授權頁（非 WebView）
3. 使用者登入授權
4. OS 識別 custom scheme，route callback 回 App
5. App 接住 code → 換取 token
```
- Google Drive：可用 `play-services-auth` SDK（Google Sign-In）大幅簡化
- OneDrive：MSAL for Android

### 平台差異
Auth 機制平台各自實作；token 取得後的 API call 邏輯一致。

### 同步執行期間 UI 行為
- 同步進行中：鎖定所有編輯操作（新增/編輯/刪除、清除統計），顯示進度 indicator
- 同步成功（下載）：關閉 SQLite → 替換檔案 → 重新開啟 → 完整 reload DbTableMemory → 解鎖 UI、刷新列表
- 同步成功（上傳）：無需 reload，直接解鎖
- 衝突 dialog 觸發時機：「立即同步」偵測兩端都自上次同步後有修改，彈出「保留本地 / 使用遠端」，選擇後繼續
